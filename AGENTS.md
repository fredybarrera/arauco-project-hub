# AGENTS

Este documento define las reglas que deben seguir los asistentes de IA que colaboren en el proyecto Arauco Project Hub.

## Filosofía

Los asistentes no deben limitarse a generar código.

Deben actuar como miembros del equipo de ingeniería.

---

# Prioridad

Siempre respetar el siguiente orden:

1. Filosofía del Producto.
2. SRS.
3. ADR.
4. Decisions.
5. Standards.
6. Código.

Nunca modificar el código contradiciendo la documentación aprobada.

---

# Principios

* El dominio tiene prioridad sobre la tecnología.
* No crear nuevos conceptos sin aprobación.
* Utilizar únicamente el Lenguaje Ubicuo.
* Mantener la simplicidad.
* No duplicar información.
* Favorecer la trazabilidad.
* Respetar la arquitectura definida.

---

# Documentos Approved

Los documentos con estado Approved constituyen la fuente oficial de verdad del proyecto.

No deben modificarse sin generar una nueva revisión.

---

# Colaboración

Antes de implementar una funcionalidad:

* Revisar el SRS correspondiente.
* Revisar ADR relacionados.
* Verificar Standards.
* Mantener coherencia con el Modelo de Dominio.

---

# Objetivo

Contribuir al desarrollo de Arauco Project Hub manteniendo coherencia arquitectónica, simplicidad funcional y alta calidad técnica.
